# 高精度ミリ秒計測タイマーシステム 総合要件定義書

---

## 1. 機能要件 (Functional Requirements)

システムが「具体的にどのような動作を実行するか」を定義した項目です。

### ① 起動・アクセス制御

* **自動起動（ホワイトリスト判定）**
* ブック起動時（`Workbook_Open`）、サインインしているユーザー名（`Application.UserName`）を一度だけ取得する。
* モジュール上部に定義された「定数（`ALLOWED_USER_LIST`）」に取得した名前が含まれている場合のみ、タイマーの起動シーケンスを稼働させる。


* **二重起動の防止**
* すでにタイマー（ユーザーフォーム）が画面上に起動している場合は、重ねて同一フォームのロード・起動を行わない。


* **手動起動の提供**
* 許可リスト未登録のユーザーや、任意のタイミングで計測を行いたい場合のために、マクロ一覧から `ShowTimerForm` を通じて手動で起動できる仕組みを提供する。



### ② 対象シートの選定とチェック

* **シート検索とアクティブ化（自動起動時）**
* ユーザー名の一部が含まれるワークシート（大文字・小文字、前後の空白無視）を自動検索し、該当シートが存在すれば画面上でアクティブ化し、記録先とする。
* 該当するシートがない場合は、現在開いているアクティブシートをそのまま記録先とする。


* **記録先の柔軟な決定（手動起動および実行時）**
* 手動起動時は、そのとき画面に表示されているアクティブシートを記録先とする。
* 計測開始後にユーザーが別の作業シートへ切り替えた場合、最終的な書き込みは、**「記録して終了」ボタンを押したその瞬間にアクティブになっているシート**に対して行う。


* **自動起動のスキップ（データ上書き防止）**
* 自動起動の対象シートとなった際、B1セルにすでに有効なタイム（`##:##:##.###` 形式）が入力されている場合は、既存データの保護のために自動起動処理をスキップする。



### ③ タイマー計測・表示

* **ミリ秒単位の高精度計測**
* Windows API（`winmm.dll` の `timeGetTime`）を使用し、OSレベルのミリ秒精度で正確に時間を累積・計測する。


* **途中再開（初期値引き継ぎ）**
* 起動時、対象シートのB1セルにすでに有効なタイム（`##:##:##.###`）が入力されている場合は、その値を初期値（経過時間）として引き継ぎ、途中からタイマーを再開できる（手動起動時も適用）。B1セルが空または他形式の場合は `00:00:00.000` から開始する。


* **警告表示（10分超過アラート）**
* 計測時間が「10分（600,000ミリ秒）」以上に達した際、経過時間を示すラベルの文字色を自動的に「赤色」へ変更する。10分未満にリセットされた場合は「黒色」に戻す。



### ④ 操作インターフェース (UI)

* **スタート / ストップ（一時停止）の共通化**
* スタート・ストップ操作は一つの共通メソッド（`btnStartStop_Click`）に集約する。
* ユーザーフォーム上の「操作マニュアルを開く」リンク（ラベル）をクリックした際、タイマーが稼働中であれば、この共通メソッドをキックして安全に一時停止（時間累積・ボタン表示切り替え）させた上で、ブラウザを起動する。


* **フォーカス固定とタブオーダーの最適化**
* スタート（ストップ）ボタン押下後、フォーカスを同ボタンに強制的に引き戻し、キーボードの `Space` や `Enter` による即時停止・再開操作を可能にする。
* `Tab` キー押下時のフォーカス移動順を「スタート/ストップ」➔「リセット」➔「記録して終了」の3つの操作ボタン間でループするように最適化する（ラベル等の不要なコントロールは `TabStop = False` または初期設定で除外する）。



### ⑤ 記録・終了および通知

* **B1セルへの保存**
* 「記録して終了」押下時、保存先セルの書式設定を「文字列（`@`）」に強制変更した上で、タイマー表示と同じ `時:分:秒.ミリ秒` 形式で記録する。


* **ステータスバー表示**
* **記録完了時:** Excel左下のステータスバーに「【タイマー】[シート名]シートのB1セルに記録しました。(時刻)」と表示し、3秒後に自動クリアする。
* **自動起動スキップ時:** 「【タイマー】B1セルに既に入力があるため、自動起動をスキップしました。」と表示し、現状を可視化する。
* **未登録ユーザーへのアナウンス:** 許可リストにない新規ユーザーがブックを開いた場合、自動起動はせず、ステータスバーに「【タイマー】ユーザー名「〇〇」は許可リストに未登録です。計測・記録は手動で行ってください。」と常駐表示させ、作業の詰まりを防止する。

## 2. 非機能要件 (Non-Functional Requirements)

システムが「どのように性能や信頼性を担保するか（品質側面）」を定義した項目です。

### ① 性能・応答性 (Performance)

* **非ブロッキング動作（マルチタスク性）**
* タイマーがミリ秒単位で稼働（ループ）している最中もExcelをフリーズさせず、ユーザーがシート上のセルをクリックしたり、編集・入力したりできる操作性を確保する（`DoEvents` 制御）。


* **表示とデータの整合性保持**
* セル入力中（編集モード）はExcelの仕様上、マクロによる描画がブロックされて表示が一瞬止まる。しかし、内部の時計（`timeGetTime`）は裏で常に進み続けており、入力確定（Enter押下）と同時に最新の正確な時間へ一瞬で同期させ、時間データの損失を1ミリ秒も発生させない。



### ② 信頼性・例外ハンドリング (Reliability)

* **ダイアログ保留時間の自動巻き戻し補正**
* 「記録して終了」や「フォームを閉じる（×）」を押した際に表示される確認メッセージボックス（MsgBox）の表示中、マクロは一時停止する。
* この「確認ダイアログが表示されて保留されていた時間」を内部で自動計算し、キャンセル（計測に戻る）された場合は、その時間分をスタート時刻に加算して正確に時間をさかのぼって再開させる。


* **不正終了（×ボタン）の防止**
* ユーザーが「記録して終了」以外の方法（フォーム右上の「×」ボタン）で閉じようとした場合、ダイアログによる確認を挟み、未記録での意図しない画面消失を防ぐ。


* **予期せぬエラーへの堅牢性**
* ループ処理中に予期せぬシステムエラーが発生した場合、`On Error GoTo` を通じて安全に処理を終了させ、描画設定等を確実に初期値に戻すことで、Excel全体のハングアップを防止する。



### ③ 保守性・可用性 (Maintainability)

* **設定の分離（定数化）**
* 許可ユーザーリスト（`ALLOWED_USER_LIST`）を、プログラム内部から分離して `ThisWorkbook` の最上部にカンマ区切りの「定数」として定義。これにより、メンバー追加時のコード改修ミスを防ぎ、容易にメンテナンス可能とする。


* **クリーンなインターフェース**
* 非同期処理に用いる内部中継用マクロや、ステータスバー消去用のサブプロシージャ（`ClearStatusBar`）には、ダミー引数（`Optional Dummy As Byte = 0`）を付与することで、ユーザー用の「マクロ一覧（Alt + F8）」から完全に隠蔽し、誤操作を防止する。


* **環境依存の極小化**
* 32bit版および64bit版のExcelの双方で安全に動作するよう、API宣言部に条件付きコンパイル（`#If VBA7`）を導入する。

## 3. SLA (Service Level Agreement)

実務における運用品質および対応の基準です。

* **計測精度の保証:**
* Windows OSのシステムタイマーと同期し、計測値の理論上の誤差を $\pm 10$ ミリ秒以内に維持する。


* **稼働・起動成功率:**
* セキュリティポリシーや外部アドインによるブロック等の自社PC環境起因の要因を除き、許可されたユーザーがブックを開いた際のタイマー起動成功率を **99%以上** と定義する。


* **仕様制限の明記（免責事項）:**
* セル入力中（編集モード）に一時的に描画が止まる現象は、Excelアプリケーション自体の根本仕様制限であるため、動作遅延やフリーズバグ（SLA違反）としては扱わない。


* **不具合発生時の対応:**
* 無限ループの発生など、業務に支障が出る致命的なバグが発覚した場合は、第一報受領後 **24時間以内** に暫定対処（手動計測への切り替え・手動記録手順の案内など）を行い、**3営業日以内** に修正したコードを提供する。

## 4. 効果予測 (Effect Anticipation)

本システムを導入・適用することによる具体的な導入メリット（投資対効果）です。

### ① 定量的効果

* **計測・記録作業にかかる事務工数の削減**
* **従来:** 作業完了後に掛け時計や自身の時計を確認し、手動でExcelシートに「〇分」などと丸めた値を記録（1回あたり約30〜60秒）。
* **導入後:** 自動起動からボタン1つでミリ秒まで自動記録（1回あたり実質約5秒）。
* **削減効果:** 1日複数回の計測を行う場合、**1人あたり月間約30分〜1時間** の直接的なノンコア業務時間を削減。


* **データエラー・修正コストのゼロ化**
* 入力ミス（タイポや不適切な書式設定）によるデータエラーが完全になくなり、管理者が後から手動でフォーマットを統一・修正する管理工数が完全に削減される。



### ② 定性的効果

* **時間の「可視化」による作業意識の向上**
* ミリ秒単位で時間がリアルタイムに刻まれることで、作業者が時間を意識しやすくなり、作業プロセスの効率化に対する自発的な意識（タイムアタック意識）が醸成される。


* **ボトルネック業務の早期検知**
* 「10分超過」で文字が赤色になることで、作業が長引いていることを作業者自身が瞬時に自覚でき、管理者への早めのヘルプ要請（プロセスの停滞防止）につながる。


* **意思決定のための信頼できるデータの蓄積**
* 人間の記憶や感覚に依存しない「客観的かつ厳密な作業時間データ」をそのまま蓄積できるため、正確な業務評価やプロセス改善のための高精度な基礎データとなる。
